# CLI Runner — Production-Ready Public Project Plan

---

## Executive Summary

- **Existing foundation is solid.** `runner.py` + `broker/engine.py` already implement loop control, status parsing, ANSI cleanup, and Windows command resolution for Codex. The refactor to multi-agent is additive, not a rewrite.
- **Adapter pattern is the right architecture.** Each agent (Codex, Gemini, Claude) gets a thin adapter implementing one interface: `resolve_cmd()`, `build_argv()`, `detect_installed()`, `install()`. The loop engine stays agent-agnostic.
- **`--agent codex|gemini|claude` is the only v1 UX change** to the main path. Existing `--codex-cmd` and `codex-runner` aliases are preserved as deprecated shims.
- **Setup is the highest-risk surface.** Installing CLIs on behalf of users requires explicit consent, dry-run previews, admin escalation warnings, and rollback guidance — not silent execution.
- **Windows-first CI on GitHub Actions** using `windows-latest` runners with Python 3.11/3.12 matrix is achievable and must be the gate, not an afterthought.
- **PyPI v1 release requires** a clean package rename (`cli-runner`), SBOM-friendly metadata, a `CHANGELOG.md`, and a `trusted publisher` GitHub Actions workflow (no API tokens in CI secrets).
- **Security model:** cli-runner never executes agent output as shell commands. It passes prompts as argv to whitelisted, user-installed binaries. The install flow shows exact commands before running them.
- **Multi-agent orchestration is explicitly post-v1.** The roadmap carries it as a named milestone so contributors understand the design constraint.
- **30-day target: working `--agent` flag + tests passing on Windows CI. 60-day: PyPI release. 90-day: polish + community docs.**

## 1. Adapter-Based Agent Architecture

### Core Interface

```
src/cli_runner/
  adapters/
    __init__.py          # exports registry
    base.py              # AgentAdapter ABC
    codex.py             # CodexAdapter
    gemini.py            # GeminiAdapter
    claude.py            # ClaudeAdapter
  runner.py              # loop engine — agent-agnostic
  setup_agent.py         # detection + install flow
  main.py                # CLI entrypoint
```

### `AgentAdapter` ABC (`adapters/base.py`)

```python
from abc import ABC, abstractmethod
from dataclasses import dataclass
from pathlib import Path

@dataclass(frozen=True)
class InvocationSpec:
    argv: list[str]          # full argv passed to subprocess
    env_overrides: dict[str, str]  # any env vars to inject

class AgentAdapter(ABC):
    name: str                # "codex" | "gemini" | "claude"

    @abstractmethod
    def resolve_cmd(self) -> list[str] | None:
        """Return resolved argv prefix (e.g. ['codex', 'exec']) or None if not found."""

    @abstractmethod
    def build_invocation(self, prompt: str, resolved_cmd: list[str]) -> InvocationSpec:
        """Build full subprocess invocation for one prompt."""

    @abstractmethod
    def is_installed(self) -> bool:
        """Fast check — shutil.which or registry probe."""

    @abstractmethod
    def install(self, dry_run: bool = False) -> bool:
        """Execute or print the install command. Returns True on success."""

    def post_install_verify(self) -> bool:
        """Re-run is_installed() after install. Default impl is sufficient."""
        return self.is_installed()
```

### Adapter Registry

```python
# adapters/__init__.py
from .codex import CodexAdapter
from .gemini import GeminiAdapter
from .claude import ClaudeAdapter

REGISTRY: dict[str, type[AgentAdapter]] = {
    "codex":  CodexAdapter,
    "gemini": GeminiAdapter,
    "claude": ClaudeAdapter,
}

def get_adapter(name: str) -> AgentAdapter:
    cls = REGISTRY.get(name.lower())
    if cls is None:
        raise ValueError(f"Unknown agent '{name}'. Choose: {', '.join(REGISTRY)}")
    return cls()
```

### Command Resolution (Windows-first)

Each adapter's `resolve_cmd()` calls a shared helper — preserve the existing `.cmd` preference logic from `broker/engine.py`:

```python
# shared util — already exists in engine.py, extract to utils/cmd.py
def resolve_windows_cmd(candidates: list[str]) -> list[str] | None:
    for name in candidates:
        # prefer .cmd shim (npm-installed tools on Windows)
        for suffix in (".cmd", ".exe", ""):
            found = shutil.which(name + suffix)
            if found:
                return [found]
    return None
```

### Prompt Invocation & Streaming

The existing `_run_codex_once` in `runner.py` is the right pattern. The loop engine calls `adapter.build_invocation(prompt)` and passes `spec.argv` to `subprocess.Popen`. Streaming is line-buffered stdout+stderr merged — no PTY needed for v1 (PTY is already handled by the broker engine for interactive mode; the simple subprocess path is sufficient for loop invocation).

### Exit / Status Handling

| Return Code | Meaning |
|---|---|
| 0 | `RUN_STATUS:DONE` verified |
| 1 | Usage/config error |
| 2 | `RUN_STATUS:REWORK` |
| 3 | Max loops exhausted (`CONTINUE`) |
| 130 | SIGINT / KeyboardInterrupt |
| 4 | Agent not installed |
| 5 | Agent install failed |

## 2. Setup / Install Strategy (Windows)

### Detection Phase

```
cli-runner setup [--agent codex|gemini|claude|all]
```

1. Call `adapter.is_installed()` for each selected agent.
2. Print a status table:
   ```
   codex    [FOUND]   C:\Users\you\AppData\Roaming\npm\codex.cmd
   gemini   [MISSING]
   claude   [FOUND]   C:\Users\you\AppData\Local\...
   ```
3. For each MISSING agent, show the install command that will be run and ask for confirmation.

### Install Execution

```
Would install gemini using:
  winget install Google.GeminiCLI
  -- OR --
  npm install -g @google/gemini-cli

Proceed? [y/N]:
```

- Never install silently. Always show the exact command.
- `--yes` / `-y` flag skips confirmation (for CI/scripted use — must be explicit).
- `--dry-run` prints commands without executing.
- Run install in a subprocess with visible output (not captured).
- Detect if command requires admin elevation (UAC) and warn: `"This may require an elevated terminal."`

### Rollback / Error Behavior

- cli-runner does **not** attempt uninstall on failure — package managers own that.
- On failure: print the exit code + suggest manual install URL from adapter metadata.
- Post-install verification: re-run `is_installed()` after install subprocess completes. If still not found, print `"Install reported success but binary not found on PATH. You may need to restart your terminal or add to PATH manually."`

### Adapter Install Specs (v1 known installs)

| Agent | Primary install | Fallback |
|---|---|---|
| codex | `npm install -g @openai/codex` | winget (if published) |
| gemini | `npm install -g @google/gemini-cli` | `pip install gemini-cli` (if published) |
| claude | `npm install -g @anthropic-ai/claude-code` | winget (if published) |

These are stored as data in each adapter, not hardcoded in setup logic.

## 3. CLI UX Proposal

### Primary Command

```
cli-runner [OPTIONS] "task text"
```

### Flags

| Flag | Default | Notes |
|---|---|---|
| `--agent codex\|gemini\|claude` | `codex` | Selects adapter |
| `--max-loops N` | 50 | Also `RUNNER_MAX_LOOPS` env |
| `--strict-completion / --no-strict-completion` | on | Completion gate |
| `--yes` / `-y` | off | Skip confirmations (setup) |
| `--dry-run` | off | Show but don't execute (setup) |
| `--codex-cmd CMD` | `codex exec` | Deprecated shim, still works |
| `--version` | — | Print version + exit |

### Subcommands

```
cli-runner setup [--agent all|codex|gemini|claude] [--yes] [--dry-run]
cli-runner status                          # print detected agents + versions
cli-runner "your task here"                # primary use
```

### Examples

```powershell
# Run task with Codex (default)
cli-runner "implement the auth module"

# Run with Claude adapter
cli-runner --agent claude "implement the auth module"

# Run with Gemini, 20 loop cap
cli-runner --agent gemini --max-loops 20 "refactor the data layer"

# Check what's installed
cli-runner status

# Install missing agents with confirmation
cli-runner setup --agent all

# Silent install for CI
cli-runner setup --agent codex --yes

# Preview install without running
cli-runner setup --agent gemini --dry-run

# Pipe task from file
cat task.txt | cli-runner --agent claude
```

### Backward Compatibility

- `codex-runner "task"` still works (entry point alias, warns deprecation to stderr).
- `--codex-cmd` still works but prints: `"[deprecated] --codex-cmd is superseded by --agent codex. Use --agent codex to configure the Codex adapter."`.

## 5. Security and Trust Model

### What cli-runner Does

- Reads a prompt string from the user.
- Passes it as a positional argument to a **user-installed, whitelisted binary** (codex/gemini/claude).
- Never interprets or shells-out that prompt string itself.
- Never executes agent output as a shell command.

### Threat Model

| Threat | Mitigation |
|---|---|
| Prompt injection via task text | cli-runner is a dumb pipe — it's the agent's responsibility. We doc this clearly. |
| Malicious `--codex-cmd` / `--agent` override | Only registered adapter names accepted; `--codex-cmd` deprecated in v1 |
| Install of unintended packages | Show exact command + require `y` confirmation; `--dry-run` always available |
| PATH hijacking (fake `codex.cmd`) | Adapters resolve with `shutil.which` — same trust as running `codex` manually |
| Supply chain (cli-runner itself) | Trusted publisher OIDC release, signed tags, `SBOM` in release artifacts |
| Admin escalation abuse | Warn when install likely needs elevation; never call UAC APIs directly |

### What We Document

- cli-runner does not sandbox or inspect agent output.
- Users are responsible for the agents they choose to run.
- `--yes` in scripts is safe only when the install source is understood.
- SECURITY.md with a GitHub private disclosure URL.

### Fake Agent Pattern (already exists at `tests/fixtures/fake_agent.py`)

Extend `fake_agent.py` to support `--agent-name` so the same fixture covers all three adapters. The fake agent script reads argv, emits configured output (DONE/CONTINUE/REWORK), exits with configured code.

### Windows-Specific Test Concerns

- Test `.cmd` resolution: verify the adapter finds `codex.cmd` when `codex.cmd` is on PATH but `codex` is not.
- Test encoding: subprocess output via `cp1252` fallback when `utf-8` fails.
- Test PATH separator: ensure no `/`-vs-`\` bugs in `shutil.which` results.
- Test admin warning: verify setup prints elevation warning when install cmd is `winget`.
